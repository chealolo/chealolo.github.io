우여곡절 hexo 블로그 만들기!

2018-10-07
#Hexo

Github hexo 블로그 만들기 !

IT회사에 처음 입사한후 많은 변화가 생겼다.
일단은 블로그를 시작한점과 Javascript , 중국어 공부를 시작했다는 점이다!
사수의 권유로 블로그를 만들게 되었는데 GitHub는 처음이라 굉장히 많은 어려움을 겪었다..

ERROR ERROR ERROR!!

처음에는 지킬테마로 시작해보았는데 스타일이 들어오지않아 실패하고
두번째에는 _config.yml 파일과 package.json 파일을 삭제해버리고..
세번째는 index.html 파일이 없어지는 사태가 발생해버렸다 ..!
이러한 시행착오를 겪고나서 극복한결과 테마 적용과 글쓰는것에 성공했다 !

왜 Hexo 인가?

블로그 프레임워크 비교

  1. Hexo
  2. Hugo
  3. Jekyll

  • Hexo

    • Javascript기반으로 만들어졌다.
    • 마크다운 지원

  • Hugo

    • Go 기반으로 만들어 졌다.

  • Jekyll

    • Ruby기반으로 만들어졌다.

사실상 이제막 시작해서 뭐가뭔지 어리둥절한 상태이다. Jekyll 로 처음에 만들다가 실패해서 사수가 Hexo를 추천해주셨다. 세가지의 프레임워크는 다 각각의 장단점이 있는듯 하다.

Hexo 설치후 테마 바꾸기 !

나같은 경우는 회사에서 npm을 사용하지않고 yarn을 사용해서 yarn으로 설치를 했다.
헷갈리시는 분들을 위해서 npm , yarn 비교 명령어 사이트 링크를 걸어두겠다.

npm , yarn 명령어 비교 사이트 :
https://shift.infinite.red/npm-vs-yarn-cheat-sheet-8755b092e5cc
npm , yarn 비교 사이트 :
https://sir.kr/so_ok365/539

$ yarn add -g hexo -cli
$ hexo init 본인폴더명 지정
$ cd 본인이 지정한 폴더명
$ yarn install



이렇게 만들면 직접 만든 기본 블로그를 로컬서버로 확인해볼수 있다.
http://localhost:4000

서버에 블로그가 올라간것을 확인하고 잘올라와있으면 일단 Hexo블로그 만들기 성공 !!
이제 블로그 안에 들어갈 이름과 몇가지 수정을 해야한다.

이걸 보고 _config.yml 파일을 수정 하면 된다.
처음 테마를 보다가 나는 clean-blog 테마가 마음에 들어서 적용을 하게 되었다.

clean-blog 홈페이지: https://github.com/klugjo/hexo-theme-clean-blog
깃허브에 들어간후 clone 을 하고 폴더 이름을 clean-blog로 바꿔주면 된다.

clone에 대한 설명: http://webclub.tistory.com/317

폴더명 변경후 themes / clean-blog / _config.yml
에 들어간후 밑에 있는 코드를 보면서 수정하면 된다.

이렇게 설정을 한후 다시 로컬 서버에서 확인을 해준다.
확인은 http://localhost:4000 링크로 들어가서 하면 된다.

블로그 테마가 바뀐것을 확인한후

배포하면 Hexo 블로그 만들기 성공 !
